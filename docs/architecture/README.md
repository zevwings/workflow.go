# 架构文档

本目录用于存放 Workflow CLI 各个功能模块的架构文档。

## 📋 说明

架构文档用于记录和说明各个功能模块的设计思路、实现方法、组件结构、依赖关系等关键信息，帮助开发者理解模块的内部实现和设计决策。

## 📝 文档规范

### 模板使用

每个功能模块的架构文档应参考以下模板编写：

**模板位置**：`docs/templates/architecture/architecture.template`

该模板提供了标准的架构文档结构，包括：

- **概述**：模块定位、功能描述、统计信息
- **Lib 层架构**：核心业务逻辑层的设计
- **Commands 层架构**：命令封装层的实现
- **集成关系**：模块间的协作方式
- **核心功能**：主要功能点和使用示例
- **设计模式**：采用的架构模式和设计原则
- **错误处理**：错误处理策略和容错机制

### 文档命名

模块架构文档应使用模块名称命名，例如：
- `config.md` - 配置模块架构文档
- `prompt.md` - 提示交互模块架构文档
- `http.md` - HTTP 客户端模块架构文档

### 文档位置

- 模块架构文档应存放在 `docs/architecture/` 目录下
- 每个模块对应一个独立的架构文档文件

## 🎯 使用指南

### 创建新模块架构文档

1. 复制模板文件：
   ```bash
   cp docs/templates/architecture/architecture.template docs/architecture/{模块名}.md
   ```

2. 根据模板中的占位符（`{模块名}`、`{功能描述}` 等）填写实际内容

3. 确保文档包含以下关键信息：
   - 模块的定位和职责
   - Lib 层和 Commands 层的结构
   - 核心组件和设计模式
   - 模块间的依赖和集成关系
   - 使用示例和扩展指南

### 更新现有架构文档

当模块实现发生变化时，应及时更新对应的架构文档，确保文档与代码实现保持一致。
